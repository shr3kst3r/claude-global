# KBA Structure Templates

Use these as starting points. Adapt sections freely based on the topic.

## How-To / Procedural

Best for: installation guides, configuration walkthroughs, setup procedures.

```
# [Title]

## Table of Contents

## Overview
Brief description of what this article covers and why it matters.

## Prerequisites
- Required knowledge, tools, or access

## Step-by-Step Instructions

### Step 1: [Action]
[Explanation + commands/code]

### Step 2: [Action]
...

## Verification
How to confirm everything is working correctly.

## Troubleshooting
Common issues and their resolutions.

## References
Links to official docs, related KBAs, further reading.
```

## Conceptual / Explainer

Best for: architecture overviews, technology comparisons, design decisions.

```
# [Title]

## Table of Contents

## Overview
What this concept is and why it matters.

## Background
Historical context or foundational knowledge needed.

## Key Concepts

### [Concept 1]
[Explanation with diagrams or examples]

### [Concept 2]
...

## How It Works
Detailed explanation of the internals or mechanics.

## Practical Examples
Real-world scenarios demonstrating the concept.

## Trade-offs and Considerations
Pros, cons, and when to use or avoid this approach.

## References
```

## Troubleshooting / Diagnostic

Best for: debugging guides, incident runbooks, error resolution.

```
# [Title]

## Table of Contents

## Overview
What problem space this covers.

## Symptoms
How to recognize the issue.

## Root Causes
Common causes ranked by likelihood.

## Diagnostic Steps

### Step 1: [Check]
[Commands and expected output]

### Step 2: [Check]
...

## Resolution

### Fix for [Cause 1]
...

### Fix for [Cause 2]
...

## Prevention
How to avoid this issue in the future.

## References
```

## Reference / Cheat Sheet

Best for: command references, API summaries, configuration options.

```
# [Title]

## Table of Contents

## Overview
What this reference covers and how to use it.

## Quick Reference
Most commonly needed items in a compact table or list.

## Detailed Reference

### [Category 1]
| Item | Description | Example |
|------|-------------|---------|
| ...  | ...         | ...     |

### [Category 2]
...

## Examples
Practical usage examples for common scenarios.

## References
```
